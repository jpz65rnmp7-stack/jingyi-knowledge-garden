# 人生最贵的错误——把不确定当确定来决策

> 确定的世界里，最优解只有一个。不确定的世界里，最优解是三个字：活下来，然后调整。

## 两种决策，用错了工具

人类喜欢确定性。不喜欢"我不知道"。

所以大多数人在不确定的情况下——用了应对确定的工具。

| 确定型决策 | 不确定型决策 |
|-----------|------------|
| 有标准答案 | 没有标准答案 |
| 信息完备 | 信息不完备 |
| 结果可预测 | 结果不可预测 |
| 一次做对 | 做→看→调 |
| 数学公式 | 试错迭代 |

**用确定型工具处理不确定的问题 = 给自己一个虚假的安全感，然后撞得头破血流。**

## 贝叶斯更新：最好的决策是"随时可以改的决策"

贝叶斯定理的核心：**不做一次性的终极判断。做一个初始判断，然后用新信息不断更新它。**

初始判断（先验概率）→ 新信息来了 → 更新判断（后验概率）→ 更多信息 → 继续更新。

| 贝叶斯更新 | 非贝叶斯 |
|-----------|---------|
| "我认为是 A，但在新信息面前我愿意改" | "我认为是 A，别跟我说别的" |
| 错了是学习机会 | 错了是失败 |
| 越更新越准确 | 越坚持越离谱 |

**最贵的错误不是"判断错了"。是"判断错了但坚持不改"。**

## 情景规划：不只准备 Plan A

传统规划：想一个目标 → 做一个计划 → 执行。

情景规划：**你想 3-5 个可能的未来 → 每个都准备一个方案 → 选触发条件 → 看现实往哪边走。**

| 步骤 | 做什么 |
|------|--------|
| 1. 识别关键不确定性 | 什么因素决定了未来的走向？ |
| 2. 构建情景 | 最好的情况·最坏的情况·最可能的情况·意外情况 |
| 3. 每个情景做方案 | 不是"Plan B 备着"——是每个情景都有正经方案 |
| 4. 定触发条件 | 什么信号出现时，切换到哪个方案？ |
| 5. 等待信号，果断切 | 不要心疼 Plan A——它本来就是一个假设 |

**情景规划不是为了预测未来。是为了让你在任何未来里都有牌可打。**

## 敏捷迭代：把大赌注变小

很多人把人生决策当成了赌场——"选这条路，押上所有筹码"。

聪明人把人生决策当成实验室——"先做一个小实验，看看结果，再决定要不要加注。"

| 赌场模式 | 实验室模式 |
|---------|-----------|
| 辞职创业 | 周末先做三个月，验证有人付钱再辞 |
| 押上全部积蓄开一个店 | 先摆摊/快闪验证产品和位置 |
| 结婚 | 先同居 |
| 上四年大学确定方向 | 先上一学期，喜欢的继续，不喜欢的换 |

**不确定性决策的第一原则 = 不要一次性押上全部。**

## 景一的冶炼

| 方法 | 一句话 |
|------|--------|
| 贝叶斯更新 | 随时准备改 |
| 情景规划 | 多个未来都有牌 |
| 敏捷迭代 | 小实验→大决策 |
| 底线思维 | 最坏情况能不能承受 |

**这个世界不奖励"预测对了"的人。这个世界奖励"不管发生什么，都能继续往前走"的人。**
