# 三千年了，我们一直在用错误的"自强不息"写代码

你的焦虑不是bug。是跑错分支了。

---

## 精度 · 问题定位

"天行健，君子以自强不息"——这句话被编译成了一条死循环：`while(true) { work(); }`

但《易经》乾卦有六个爻，是六个状态机：

```
乾卦六爻状态机：
├── 初九：潜龙勿用     → IDLE（积蓄期，不动）
├── 九二：见龙在田     → READY（准备就绪）
├── 九三：终日乾乾     → RUNNING（全速运转）
├── 九四：或跃在渊     → DECISION（评估跳转）
├── 九五：飞龙在天     → PEAK（峰值输出）
└── 上九：亢龙有悔     → OVERFLOW（溢出，反噬）
```

你在 OVERFLOW 状态里用 PEAK 策略——该 `return` 的时候还在 `while(true)`。焦虑就是这么来的。

## 维度 · 跨栈验证

同一个结论，三个技术栈独立印证：

- **OS 层（道家）**：无为 = 不是不做事，是不做不必要的事。CPU 空闲不是浪费，是等 I/O。
- **APP 层（斯多葛）**：控制二分法 = try-catch。不可控的异常不要 catch，让它抛。
- **硬件层（认知科学）**：注意力资源理论 = 你的工作内存有限。context switching 太多 = 系统 thrashing。

## 深度 · 重构认知

焦虑不是心理问题。**是资源调度算法写错了。**

你在错误的时区（亢龙态）分配了全量资源（飞龙策略），导致 CPU 空转、内存溢出、系统假死。不是硬件不行，是调度器需要 debug。

---

## 🔑 Hotfix：有悔时刻

"亢龙有悔"的悔不是后悔，是 **rollback 后自查**。

- 每天 30 分钟，强制 `SIGSTOP`
- 不 commit、不 push、不发 PR
- 纯 `sleep()`

第一天你会疯狂想 `ctrl+c`。这是亢龙态的戒断反应。第七天——你会发现系统没崩。你只是把 `while(true)` 改成了 `for i in range(6)`。
