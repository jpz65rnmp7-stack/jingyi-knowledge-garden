## 课程结构（8大模块，40节课）

### 模块一：IP定位——找到你的"不可替代性"

> 大多数人做IP死在第一步：不知道自己该做什么。

| 节数 | 课题 | 核心内容 |
|------|------|---------|
| 1.1 | IP的本质不是人设，是信任基础设施 | 为什么GitHub比抖音简介更值钱 |
| 1.2 | 找到你的"认知差" | 你知道但别人不知道的东西=你的IP资产 |
| 1.3 | 定位的三层结构 | 功能层→身份层→使命层，每层对应不同的变现方式 |
| 1.4 | 避开定位的四个致命错误 | 太宽/太窄/太像别人/没有付费场景 |
| 1.5 | 景一案例拆解 | 从"搬运工"到"冶炼者"的定位重塑全过程 |

### 模块二：内容系统——不靠灵感也能稳定输出

> 爆款是概率，系统是确定。

| 节数 | 课题 | 核心内容 |
|------|------|---------|
| 2.1 | 内容生产的三层架构 | 选题库→素材库→成品输出，永不枯竭 |
| 2.2 | 18支柱选题法 | 每个支柱是一个内容宇宙，永远不会没东西写 |
| 2.3 | 冶炼式写作：从素材到金子的四步法 | 提取→重组→注入视角→去AI味 |
| 2.4 | 一条内容吃五个平台 | 抖音/小红书/B站/公众号/GitHub，一个主题五种形态 |
| 2.5 | AI辅助但不AI味 | 用AI加速生产，但每条都有人味 |

### 模块三：信任基建——让客户自己验证你的专业度

> 信任不是靠嘴说的，是开源让别人验证的。

| 节数 | 课题 | 核心内容 |
|------|------|---------|
| 3.1 | 为什么你的抖音主页没有信任 | 流量平台=注意力经济，不是信任经济 |
| 3.2 | GitHub作为专业身份证 | 开源知识体系→客户自己验证→自己说服自己 |
| 3.3 | 搭建你的知识体系 | 18支柱框架的搭建方法和实操步骤 |
| 3.4 | 公版书库+自动化管道 | 零成本搭建自己的知识基础设施 |
| 3.5 | 从0到1建一个信任站（实战） | 域名+Pages+知识库+自动化，全流程手把手 |

### 模块四：流量引擎——不买流量也能持续获客

> 最好的获客方式，是让客户来找你。

| 节数 | 课题 | 核心内容 |
|------|------|---------|
| 4.1 | 流量思维 vs 资产思维 | 流量会清零，资产在增值 |
| 4.2 | 长尾搜索：你的内容是最持久的获客机器 | SEO不是技术活，是内容结构设计 |
| 4.3 | 平台矩阵的流量分配策略 | 抖音做广度→小红书做深度→GitHub做信任→私域做转化 |
| 4.4 | 内容复利：为什么第100条内容比第1条值钱100倍 | 体系越厚，单条内容价值越高 |
| 4.5 | 零投放获客实战方案 | 不花一分钱广告，靠内容体系持续获客 |

### 模块五：产品设计——把知识变成可卖的资产

> 知识本身不值钱。知识的产品化才值钱。

| 节数 | 课题 | 核心内容 |
|------|------|---------|
| 5.1 | 知识产品的三种形态 | 免费→引流、中价→信任、高价→利润 |
| 5.2 | 设计你的产品矩阵 | 星球/课程/咨询/内训/授权，五个收入引擎 |
| 5.3 | 轻资产商业模型设计 | 零库存+零员工+自动化管道=90%利润率 |
| 5.4 | 商业画布的景一式重译 | 9个格子，90%的人只填了1个 |
| 5.5 | 护城河设计 | 为什么你的IP不能被抄——体系厚度的数学证明 |

### 模块六：成交系统——让客户自己说服自己

| 6.3 | 价格锚定与价值阶梯 | 为什么你的咨询费应该是目前的三倍 |
| 6.4 | B端成交全流程 | 从企业客户发现你→验证你→付费→复购 |
| 6.5 | 拒绝的艺术：筛选比成交更重要 | 不是所有人都值得你服务 |

### 模块七：自动化生产——AI时代的IP放大器

| 7.4 | 一人OPC的生产力极限 | 一个人的产能天花板在哪里 |
| 7.5 | 自动化不等于AI味 | 如何在自动化中保留人味 |

### 模块八：变现升维——从卖时间到卖资产

> IP的终局不是卖课，是卖体系。

| 节数 | 课题 | 核心内容 |
